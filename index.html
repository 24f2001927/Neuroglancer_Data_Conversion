System Info & Instruction Manual

Welcome to the NG Data Conversion Pipeline Control Panel. This software acts as the central control plane for volumetric brain scan conversions at the Sudha Gopalakrishnan Brain Centre, IIT Madras.

Primary Purpose

The system is created to orchestrate the conversion of extremely large, high-resolution raw binary volumetric brain scans (.raw / .nhdr files) into the multi-layer OME-Zarr format. This format structures the data into hierarchical multiresolution pyramids, allowing researchers to visualize gigabyte-scale volumes seamlessly inside Neuroglancer without crashing the client's web browser.

How to Use the Panel

  1. Check Server Statuses: The top panel contains status cards for the Raw Data Server (port 10228), the local Pipeline API Server, and the Converted Data Server (port 10229). Ensure all status bulbs are green/blue indicating they are online.
  2. Inspect Directories: The two central CRT monitors pull live feeds from the HTTP listings. The left monitor lists raw files waiting for conversion, and the right monitor lists converted .zarr directories.
  3. Fetch Metadata Sidecars: Under the files lists, use the API console endpoints to fetch .nhdr metadata text dynamically.
  4. Trigger Conversions: Conversion requests can be sent directly to the pipeline. The backend will automatically map raw scanner IDs, check NHDR properties, compute Dask chunk matrices, and convert the volumes in non-blocking background threads.
